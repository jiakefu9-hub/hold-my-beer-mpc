# 实验计划与开发路线图 (Experimental Plan & Roadmap)

本项目旨在实现人形机器人在移动过程中的单臂末端稳定控制（Hold My Beer 任务）。为了保证开发的高效性、代码的鲁棒性以及最终成果的学术/工程价值，我们制定了以下严格的“三步走”实验验证计划。

---

## 🏃 第一步：零控制基线与扰动采集（System Identification）

**实验设置：** 
左手和右手都拿着对应质量的水瓶（真机/仿真中），左右手所有关节角度全部固定在 0 度（使用底层 PD 锁死）。

**动作：** 
让 G1 机器人在设定的速度下行走（比如 0.5m/s）。

**收集数据：** 
记录上半身机座（Torso，不是骨盆）的数据（线性加速度、角速度、角加速度、姿态 ${}^W R_B$），以及机器人行走周期的相位（比如左脚触地、右脚触地的时间点）。

**核心目的（为什么要最先做这一步？）：**
1. **安全与验证**：必须先证明加上了这两个瓶子后，G1 自带的底盘行走算法依然能走稳。如果在这一步机器人都摔了，说明瓶子太重或者位置不对，需要先调整物理参数，而不是去调控制代码。
2. **纯粹的扰动**：双臂死锁时，手臂没有任何主动发力，此时记录下的机座晃动是最“纯粹”的、仅由腿部行走产生的固有扰动。
3. **MPC 的刚需（前馈补偿）**：MPC 的强大在于“预测（Prediction）”。需要用这一步收集到的躯干晃动数据，拟合出一个**周期性的扰动模型（Disturbance Model）**或者轨迹。只有拿到了这个模型，才能在未来的 MPC 目标函数中加入前馈（Feedforward）补偿项，提前抵消躯干带来的震动。

---

## ⚖️ 第二步：部署并调节 PID（建立性能基线 Baseline）

**实验设置：** 
左臂依然固定在 0 度（拿瓶子），右臂切换为 PID 控制器（拿瓶子，目标是保持末端姿态水平）。

**动作：** 
同样的速度行走。

**收集数据：** 
记录右臂末端的误差（加速度、倾斜角等）。

**核心目的（为什么排第二？）：**
1. **跑通代码链路**：PID 不需要复杂的动力学模型，写起来最快。在这一步，核心任务是跑通 G1 SDK 的闭环控制链路（读取状态 -> 计算误差 -> PID 计算力矩 -> 下发力矩），并且确保控制循环能稳定在足够高的频率（比如 500Hz 或 1000Hz）。
2. **提供对比靶子**：需要把 PID 调到“相对不错但肯定不够完美”的状态。这个状态产生的数据，就是以后用来衬托 MPC 性能优势的“垫脚石”和基准（Baseline）。

---

## 🚀 第三步：部署并验证 MPC（核心成果）

**实验设置：** 
左臂固定在 0 度（拿瓶子），右臂切换为 MPC 控制器。

**动作：** 
同等条件行走。

**实施细节：** 
此时，代码架构已经通过 PID 验证过了，底层收发没问题。只需要把 PID 的计算模块替换成 MPC 求解器（Solver）。同时，把第一步拟合出来的机座扰动模型作为观测状态（State）或前馈输入喂给 MPC。

**收集数据：** 
记录右臂末端误差，生成对比图表（与死锁状态、PID 状态进行对比）。

---

## 📈 总结与评估指标

最终的成果展示应包含以下图表：
- **末端姿态误差对比曲线**（Dead-lock vs PID vs MPC）
- **末端线加速度/角加速度对比曲线**（评估“水会不会洒出来”）
- **关节力矩平滑度对比**（验证 MPC 是否有效抑制了高频抖动）
